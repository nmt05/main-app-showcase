# AGENTS.md

## Mission
This repository hosts a website whose main goal is to demonstrate the `GameTwoShape` mobile app.
The home experience should explain the game quickly and make visitors want to try it.

## Target Audience
- Course instructors and classmates reviewing the project
- Potential players
- Potential collaborators

## Content Priorities
1. Gameplay clarity: what the player does and why it is challenging
2. Proof: real screenshots, clips, or gameplay visuals
3. Value: focus, reflex, and two-hand coordination training
4. Action: clear links to demo, repository, or download
5. Team and technology credits

## Page Rules
- Keep the site static-first with HTML, CSS, and lightweight JavaScript.
- Reuse existing assets in `images/` before adding new files.
- Remove template filler text; prefer short, scannable copy.
- Keep text encoding clean in UTF-8 to avoid broken Vietnamese characters.
- Mobile-first layout is required.
- Keep accessibility baseline:
  - meaningful `alt` text for images
  - visible focus states and readable color contrast
  - semantic headings and section structure
- Keep the primary CTA above the fold on hero sections.

## Recommended Section Order
1. Hero with value proposition and CTA
2. Gameplay loop (how it works)
3. Core features and difficulty progression
4. Visual proof (gallery/video)
5. Tech stack and team
6. Final CTA and contact/repository links

## Editing Guardrails
- Do not delete `index.html` or `index-demo.html` unless explicitly requested.
- New alternatives (for example `index2.html`) should be self-contained and easy to compare.
- Keep dependencies minimal and avoid heavy frameworks for simple showcases.
- Keep social and repository links current when available.

## Definition Of Done For Website Changes
- Page loads without console errors
- Layout is usable on mobile and desktop
- Buttons and links are meaningful or clearly marked as placeholders
- Messaging stays aligned with the mission: demonstrate the mobile game
